35 /*
36  * The console utilities service provides support for serial input and output (even from
37  * an interrupt context) by employing the UART adapter for reading and writing, and through
38  * one of the UART peripheral HW instances.
39  *
40  * When attempting to capture serial input or print a string from within an application task
41  * or interrupt, the libC standard _read()/_write() are redirected to console_read()/write().
42  * To allow these reads/writes, console uses it's own task (console_task_func()) that does
43  * the actual UART hardware access through the UART adapter (ad_uart_read/write_async()).
44  * Synchronization between the console task and the main execution domain is achieved via an
45  * OS-based set of events and notifications.
46  *
47  * Notes:
48  * - Console does not use additional RAM for printing. RAM that is used is allocated during
49  * initialization only.
50  * - If data flow is too fast for UART, calls from task will wait, while calls from interrupt
51  * may lose some data.
52  * - When the console service and CONFIG_RETARGET are both enabled, the serial input and output
53  * activity will be automatically handled by console instead of standard retarget implementation.
54  * - In case of printing, it must be assured that the HW UART CTS line/pin of the USB/UART IC
55  * converter is properly connected to the relevant M33 GPIO pin. Additionally, that an IRQ handler
56  * is defined to be triggered upon a CTS event. See the console_wkup_handler() description for more.
57  */
