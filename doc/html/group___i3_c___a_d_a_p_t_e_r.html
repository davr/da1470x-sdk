◆ CONFIG_I3C_USE_SYNC_TRANSACTIONS

#define CONFIG_I3C_USE_SYNC_TRANSACTIONS   (1)

Controls whether I3C synchronous transaction API will be used.

I3C synchronous transaction API maintains state in retention RAM. If the API is not to be used, setting this macro to 0 will save retention RAM.

bool  force

Close I3C controller.

This function:

  • Aborts ongoing transactions
  • De-initializes the drivers associated with the controller
  • Resets controller interface IOs (as specified in ad_i3c_open())
  • Releases the controller resources
Parameters
[in]handlehandle returned from ad_i3c_open()
[in]forceforce close even if controller is occupied
Returns
0: success, <0: error code

◆ ad_i3c_open()

ad_i3c_handle_t ad_i3c_open ( const ad_i3c_controller_conf_t *  conf)

Open I3C controller.

This function:

  • Acquires the resources needed for using the controller
  • Configures the controller interface IOs
  • Initializes the drivers associated with the controller
Parameters
[in]confcontroller configuration
Returns
!=0: handle that should be used in subsequent API calls, NULL: error
Note
The function will block until it acquires all controller resources
Warning
Controllers configuration conf must point to a valid ad_i3c_controller_conf_t struct till ad_i3c_close() is called.

int ad_i3c_private_read ( ad_i3c_handle_t  handle,
uint8_t *  rbuf,
size_t  rlen,
i3c_private_transfer_config *  i3c_transfer_cfg,
OS_TICK_TIME  timeout 
)

Perform a blocking private read transaction with time out.

This function performs a synchronous private read only transaction

Parameters
[in]handlehandle returned by ad_i3c_open()
[out]rbufbuffer for incoming data
[in]rlennumber of bytes to read
[in]i3c_transfer_cfgI3C transfer configuration
[in]timeouttime in OS ticks that is expected for read to complete
Returns
0: success, !0: error code

In case of transfer error this function returns response status. The following macro definitions could be used to parse response in order to check error status, transaction ID and remaining data length if the transfer terminated early.

See also
HW_I3C_RESPONSE_PORT_DATA_LEN, HW_I3C_RESPONSE_PORT_ERR_STATUS, HW_I3C_RESPONSE_PORT_TID
ad_i3c_open()
Warning
In DMA mode the supplied buffer address and the transfer length must be word-aligned.
In case of AD_I3C_ERROR_CONTROLLER_ABORT_FAIL the adapter should close(ad_i3c_close()) and reopen(ad_i3c_open).

Initiate a non blocking private read transaction.

This function initiates an asynchronous private read only transaction Caller task should retry until function returns no error Callback will be called when transaction is completed

Parameters

[out]rbufbuffer for incoming data (rbuf buffer should not be touched till callback is called)
[in]rlennumber of bytes to read
[in]i3c_transfer_cfgI3C transfer configuration
[in]cbcallback to call after transaction is over (from ISR context)
[in]user_datauser data passed to cb callback
Returns
0 on success, <0: error

The response status is returned by callback function. The following macro definitions could be used to parse response in order to check error status, transaction ID and remaining data length if the transfer terminated early.
